**New properties on ClaudeMonitor:**

```swift
let ttsService = TTSService()
private static let summaryFile = ipcDir + "/last_summary.txt"
private var summarySource: DispatchSourceFileSystemObject?
private var lastSummaryHash: Int = 0
```

**New method: setupSummaryWatcher()**
Port from LoopEngine.watchFile() / startWatching():
1. Ensure the summary file exists (create if not): `FileManager.default.createFile(atPath:contents:)`
2. Read initial hash: `if let data = FileManager.default.contents(atPath: Self.summaryFile) { lastSummaryHash = data.hashValue }`
3. Open with `open(Self.summaryFile, O_EVTONLY)`, create `DispatchSource.makeFileSystemObjectSource` with `.write, .extend, .rename, .delete` on `.main` queue
4. Event handler calls `onSummaryFileChanged()`
5. Cancel handler closes fd
6. Resume source, store in `summarySource`

**New method: onSummaryFileChanged()**
Port from LoopEngine.onSummaryChanged():
1. Read file contents, validate non-empty
2. Compare hash — skip if unchanged (`guard hash != lastSummaryHash`)
3. Update `lastSummaryHash = hash`
4. Guard: `guard autoSpeak, !isRecording else { return }` — NEVER interrupt recording
5. Create Task:
   ```swift
   Task {
       let summary = await ttsService.speakSummary(text)
       await MainActor.run {
           guard !self.isRecording else { return }  // re-check after async summarization
           self.lastSummary = summary
       }
   }
   ```

**Wire into init():**
- Call `setupSummaryWatcher()` after `setupFileWatcher()` (before isActive assignment)

**Wire into isActive didSet:**
- When `isActive = true`: call `setupSummaryWatcher()` in the restart block
- When `isActive = false`: `summarySource?.cancel(); summarySource = nil; ttsService.stopSpeaking()`

**Wire into deinit:**
- Add `summarySource?.cancel()`
- Add `ttsService.stopSpeaking()`

**CRITICAL — recording priority in processEvent "stop" case:**
- In the existing `case "stop":` block, after `playDoneChime()`, do NOT trigger auto-speak here. The summary hook writes the file, which triggers the file watcher. The event flow is: Claude stops -> hook writes file -> watcher fires -> onSummaryFileChanged runs. No duplicate trigger needed.

**Mirror isSpeaking for UI (optional but useful):**
Add `@Published private(set) var isSpeaking: Bool = false` and in init, add:
```swift

    .receive(on: DispatchQueue.main)
    .assign(to: &$isSpeaking)
```

2. In `toggle()`, before the `else` branch that calls `startRecording()`, add TTS cutoff:
   ```swift
   func toggle() {
       if isRecording {
           stopRecording()
       } else {
           // Cut TTS immediately — recording has absolute priority
           if let tts = ttsService, tts.isSpeaking {
               tts.stopSpeaking()
           }
           startRecording()
       }
   }
   ```

3. In `startRecording()`, add TTS kill as first action (same as LoopEngine):
   ```swift
   // Recording has absolute priority — kill TTS first
   if let tts = ttsService, tts.synthesizer.isSpeaking {  // check synthesizer directly

       usleep(300_000)  // wait for audio session to fully release
       log("killed TTS before recording")
   }
   ```
   NOTE: TTSService.synthesizer is private. Instead, check `tts.isSpeaking` (the @Published property) and call `tts.stopSpeaking()`. The 300ms wait is important for audio session release.

   Actually, simpler approach: just check `tts.isSpeaking` and call `tts.stopSpeaking()` — the stopSpeaking method already has a 100ms usleep. Add an extra `usleep(200_000)` after for the full 300ms the prototype uses.
