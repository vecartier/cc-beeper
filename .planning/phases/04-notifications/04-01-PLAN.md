1. Create `Sources/NotificationManager.swift` with:
   - `import Foundation` and `import UserNotifications`
   - `final class NotificationManager: NSObject, UNUserNotificationCenterDelegate`
   - Private `let center = UNUserNotificationCenter.current()`
   - In `override init()`: set `center.delegate = self`, call `registerCategories()`
   - `func requestPermission()`: call `center.requestAuthorization(options: [.alert, .sound]) { _, _ in }`
   - Implement `userNotificationCenter(_:willPresent:withCompletionHandler:)` delegate method — call `completionHandler([.banner, .sound])`. This is CRITICAL: without it, macOS suppresses banners for foreground apps (this app is always foreground as a menu bar app).
   - `private func registerCategories()`: register three categories with identifiers `"PERMISSION_REQUEST"`, `"SESSION_DONE"`, `"TOOL_ERROR"` — empty actions and intentIdentifiers
   - `func sendPermissionRequest(tool: String, summary: String)`: title "Claude Needs Your Permission", body "\(tool): \(summary)", category "PERMISSION_REQUEST"
   - `func sendSessionDone()`: title "Claude Is Done", body "Session completed.", category "SESSION_DONE"
   - `func sendToolError(tool: String)`: title "Claude Tool Error", body "\(tool) encountered an error.", category "TOOL_ERROR"
   - `func sendPermissionTimeout()`: title "Permission Timed Out", body "A permission request was not answered in time.", category "TOOL_ERROR"
   - `private func send(id: String, categoryId: String, title: String, body: String)`: creates `UNMutableNotificationContent` with title/body/sound(.default)/categoryIdentifier, uses `UNTimeIntervalNotificationTrigger(timeInterval: 0.1, repeats: false)` (NOT nil trigger — inconsistent behavior on macOS), UUID-based identifier to avoid replacing previous notifications
   - Each public send method generates its own UUID-based id: `"perm-\(UUID().uuidString)"`, `"done-\(UUID().uuidString)"`, `"err-\(UUID().uuidString)"`, `"timeout-\(UUID().uuidString)"`

   ```bash
   codesign --force --deep --sign - Claumagotchi.app
   echo "Signed Claumagotchi.app (ad-hoc)"
   ```
   This is REQUIRED — without code signing, `requestAuthorization` silently returns `granted = false` and no permission dialog appears.

1. **New property** (alongside `soundEnabled` and `autoAccept`):
   ```swift
   @Published var notificationsEnabled: Bool {
       didSet { UserDefaults.standard.set(notificationsEnabled, forKey: "notificationsEnabled") }
   }
   ```

2. **New private instance** (alongside existing private vars):
   ```swift
   private let notificationManager = NotificationManager()
   ```

3. **In `init()`** — add alongside `soundEnabled` init, BEFORE `setupFileWatcher()`:
   ```swift
   notificationsEnabled = UserDefaults.standard.object(forKey: "notificationsEnabled") as? Bool ?? true
   notificationManager.requestPermission()
   ```
   Call requestPermission() eagerly so the system dialog appears on first launch, not lazily when the first event fires (avoids missing the first notification).

4. **In `processEvent`, permission branch** (both the `type == "permission"` block at ~line 215 and the `type == "notification"` permission_prompt block at ~line 232): Inside the `else` block (the `!autoAccept` path), BEFORE `awaitingUserAction = true`, add:
   ```swift
   if notificationsEnabled {
       notificationManager.sendPermissionRequest(
           tool: pendingPermission?.tool ?? "Unknown",
           summary: pendingPermission?.summary ?? ""
       )
   }
   ```
   Note: `loadPendingPermission` is called earlier in the same block and uses async retries, so `pendingPermission` may not be populated yet. Use the nil-coalescing fallback. The notification still fires with "Unknown" which is acceptable — the permission data races with the event.

5. **In `processEvent`, `permission_timeout` branch** (~line 249): Replace the bare `return` with:
   ```swift
   if notificationsEnabled {
       notificationManager.sendPermissionTimeout()
   }
   return
   ```

6. **In `processEvent`, switch statement**: Separate `post_tool_error` from the `pre_tool, post_tool` case. Change:
   ```swift
   case "pre_tool", "post_tool", "post_tool_error":
       if !sid.isEmpty { sessionStates[sid] = .thinking }
       updateAggregateState()
   ```
   To:
   ```swift
   case "pre_tool", "post_tool":
       if !sid.isEmpty { sessionStates[sid] = .thinking }
       updateAggregateState()
   case "post_tool_error":
       if !sid.isEmpty { sessionStates[sid] = .thinking }
       updateAggregateState()
       if notificationsEnabled {
           let tool = event["tool"] as? String ?? "Tool"
           notificationManager.sendToolError(tool: tool)
       }
   ```

7. **In `processEvent`, stop branch**: After `playDoneChime()` and before `startIdleTimer(interval: 60)`, add:
   ```swift
   if notificationsEnabled {
       notificationManager.sendSessionDone()
   }
   ```
