Create Sources/SettingsView.swift with the following structure:

**SettingsView (root):**
- `import SwiftUI` and `import ApplicationServices` (for AXIsProcessTrusted)
- `@EnvironmentObject var monitor: ClaudeMonitor`
- `@EnvironmentObject var themeManager: ThemeManager`
- Body: `TabView` using the `Tab` API (NOT `tabItem()` — per SwiftUI Pro rules):
  - `Tab("General", systemImage: "gear")` containing `GeneralSettingsView()`
  - `Tab("Appearance", systemImage: "paintbrush")` containing `AppearanceSettingsView()`
  - `Tab("AI", systemImage: "cpu")` containing `AISettingsView()`
  - `Tab("Privacy", systemImage: "lock.shield")` containing `PrivacySettingsView()`
- Apply `.frame(width: 550, height: 400)` to the TabView

**GeneralSettingsView:**
- `@EnvironmentObject var monitor: ClaudeMonitor`
- `Form` layout with:
  - Section "Sound": Toggle("Enable sounds", isOn: $monitor.soundEnabled)
  - Section "Notifications": Toggle("Enable notifications", isOn: $monitor.notificationsEnabled)
  - Section "Global Hotkeys": Show current hotkey status. If `!AXIsProcessTrusted()`, show a Button("Grant Accessibility Access") that calls `AXIsProcessTrustedWithOptions` with the prompt option. If trusted, show "Option+A to allow, Option+D to deny" as a Text with `.foregroundStyle(.secondary)`.

**AppearanceSettingsView:**
- `@EnvironmentObject var themeManager: ThemeManager`
- `Form` layout with:
  - Section "Theme": A `Picker("Color Theme", selection: $themeManager.currentThemeId)` with `.pickerStyle(.radioGroup)` showing all `ThemeManager.themes` by name. Use `ForEach(ThemeManager.themes) { theme in Text(theme.name).tag(theme.id) }`.
  - Section "Mode": Toggle("Dark Mode", isOn: $themeManager.darkMode)

**AISettingsView:**
- `@EnvironmentObject var monitor: ClaudeMonitor` (not used directly here but injected for consistency)
- Local state: `@State private var selectedProvider: APIProvider = .anthropic`
- Local state: `@State private var apiKeyText: String = ""`
- Local state: `@State private var isRevealed: Bool = false`
- Local state: `@FocusState private var keyFieldFocused: Bool`
- Local state: `@State private var validationState: ValidationState = .idle` — use a local enum `ValidationState { case idle, validating, valid, invalid(String), error(String) }`
- Local state: `@State private var showDeleteConfirm: Bool = false`
- `onAppear`: Load existing key from Keychain — `apiKeyText = KeychainHelper.load(key: selectedProvider.keychainKey) ?? ""`
- Provider picker: `Picker("Provider", selection: $selectedProvider)` with `.pickerStyle(.segmented)` showing both providers. On change of selectedProvider, reload the key from Keychain for that provider.
- API key field: Swap between `SecureField` and `TextField` based on `isRevealed` state. Both use `$apiKeyText` binding and `keyFieldFocused` focus state. Eye toggle button on the right (systemImage: isRevealed ? "eye.slash" : "eye").
- Inline privacy hint: `Text("Stored in macOS Keychain").font(.caption).foregroundStyle(.secondary)` directly below the key field.
- Buttons row:
  - "Save" button: Calls `KeychainHelper.save(apiKeyText, key: selectedProvider.keychainKey)`. On success, show brief checkmark feedback by setting validationState to .valid for 2 seconds then back to .idle.
  - "Test" button: Sets validationState to .validating, calls `await APIKeyValidator.validate(key: apiKeyText, provider: selectedProvider)`, maps result to validationState. Wrap in `Task { }`.
  - "Clear" button: Sets `showDeleteConfirm = true`.
- Validation feedback: Show inline status based on validationState — ProgressView for .validating, checkmark Image for .valid, xmark with message for .invalid/.error. Use `.foregroundStyle(.green)` for valid, `.foregroundStyle(.red)` for invalid/error.
- Confirm dialog for clear: `.confirmationDialog("Remove API Key?", isPresented: $showDeleteConfirm)` with a destructive "Remove" button that calls `KeychainHelper.delete(key: selectedProvider.keychainKey)` and clears apiKeyText. Attach the dialog to the Clear button (per SwiftUI Pro navigation.md rule: attach confirmationDialog to the UI that triggers it).

**PrivacySettingsView:**
- No @EnvironmentObject needed — static content.
- `ScrollView` with `.scrollIndicators(.hidden)` (per project convention from CLAUDE.md/memory).
- VStack with:
  - Header: "Your Privacy" as a title
  - Paragraph: "Your data never leaves your Mac. We don't run servers. Everything stays local."
  - Section "API Keys": "API keys are stored in macOS Keychain, Apple's secure credential storage. They are never transmitted anywhere except directly to the provider you choose (Anthropic or OpenAI)."
  - Section "Voice Input" (future): "Voice transcription will use Apple's on-device speech recognition. No audio leaves your Mac." (Prepares for Phase 8)
  - Section "Monitoring Data": "All Claude session monitoring data stays in ~/.claude/claumagotchi/ on your local filesystem. Nothing is uploaded."
  - Use `.font(.body)` for paragraphs, `.font(.headline)` for section headers, `.foregroundStyle(.secondary)` for body text.
  - Reassuring tone throughout — per user's emphasis on "a lot of reassurance that all is local."

**Style notes (across all tabs):**
- Use `foregroundStyle()` not `foregroundColor()` — per SwiftUI Pro api.md
- Use `Form` for settings tabs (General, Appearance, AI) — standard macOS settings pattern
- Use `.padding()` for comfortable spacing within Form sections
- Do NOT use `.tabViewStyle(.sidebarAdaptable)` — let it default to top tab bar (per research pitfall 4)
